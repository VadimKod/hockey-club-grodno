## 🔧 Установка

### 1. Установи зависимости сервера

Открой PowerShell **от имени администратора** и выполни:

```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

Затем в обычном PowerShell:

```powershell
cd server
npm install
```

### 2. Настрой email (SMTP)

Скопируй `.env.example` в `.env` в папке `server`:

```env
PORT=5000
MONGODB_URI=mongodb://localhost:27017/hockey_club
JWT_SECRET=your_super_secret_jwt_key_change_this_in_production

# Email уведомления
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=your_email@gmail.com
SMTP_PASS=your_app_password
FRONTEND_URL=http://localhost:5173
```

#### Как получить пароль приложения Gmail:
1. Включи двухфакторную аутентификацию: https://myaccount.google.com/security
2. Создай пароль приложения: https://myaccount.google.com/apppasswords
3. Выбери "Почта" и своё устройство
4. Скопируй 16-значный пароль в `SMTP_PASS`

### 3. Запусти сервер

```powershell
cd server
npm run dev
```

### 4. Обнови базу данных

Удали старую базу или перезапиши коллекцию `users`, чтобы применилась новая схема ролей.

Используй seed-скрипт (если обновишь его) или создай пользователей через админ-панель.

## 📋 Использование

### Регистрация новых пользователей

При регистрации по умолчанию присваивается роль **`user2`** (пользователь 2 уровня).

### Назначение ролей

1. Войди как администратор (`admin@ledokol-grodno.by` / `Ledokol2026!Admin`)
2. Перейди в **Админ-панель** → **Пользователи**
3. Выбери роль для каждого пользователя:
   - `Пользователь 1 уровня` — для игроков и тренеров клуба
   - `Пользователь 2 уровня` — для болельщиков (роль по умолчанию)
   - `Админ` — для администраторов

### Отправка рассылок

1. Перейди в **Админ-панель** → **Рассылки**
2. Выбери тип рассылки:
   - **Пользователям 1 уровня** — расписание тренировок и матчей
   - **Пользователям 2 уровня** — предстоящие матчи

## 🎯 Проверка работы

### 1. Создай тестовых пользователей

Зарегистрируй 3 аккаунта:
- Пользователь 1 уровня: `user1@test.by` / пароль
- Пользователь 2 уровня: `user2@test.by` / пароль

В админ-панели назначь:

Войди под каждым пользователем и проверь:
- **Пользователь 1 уровня**: видит настройки "Расписание тренировок" и "Расписание матчей"

В админ-панели → Рассылки:
- Отправь тестовую рассылку пользователям 1 уровня

## 🔐 Middleware для защиты маршрутов

```javascript
const { auth, isAdmin, isUserLevel1, isUserLevel2, hasRole } = require('../middleware/auth');

// Только авторизованные
router.get('/profile', auth, (req, res) => { ... });

// Только админ
router.get('/admin', auth, isAdmin, (req, res) => { ... });

// Только пользователи 1 уровня
router.get('/training', auth, isUserLevel1, (req, res) => { ... });

// Только пользователи 2 уровня
router.get('/tickets', auth, isUserLevel2, (req, res) => { ... });

// Несколько ролей
router.get('/special', auth, hasRole(['user1', 'admin']), (req, res) => { ... });
```

## ❓ Troubleshooting

### Ошибка "npm.ps1 не может быть загружен"
```powershell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Email не отправляются
- Проверь SMTP настройки в `.env`
- Для Gmail используй пароль приложения, не обычный пароль
- Проверь логи сервера

### Роли не применяются
- Очисти кэш браузера
- Перезайди в систему (logout/login)
- Проверь, что токен обновился
